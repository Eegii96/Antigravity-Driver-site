## 2. 🔴 ЯАРАЛТАЙ — Аюулгүй байдал (1-2 долоо хоног дотор)

### S1. Бүх хэрэглэгчийн нэг ерөнхий нууц үг (`Password123!`)
- `db.ts` дотор бүх Firebase Auth нууц үгийг `'Password123!'` болгож нэгтгэдэг.
- **Үр дагавар:** И-мэйл мэдэж байгаа хэн ч `Password123!`-ээр **дурын аккаунтад нэвтэрнэ.** Нэг хэрэглэгчээр нэвтэрч өөрийн үнэлгээ, түүхийг хуурамчаар засах боломжтой.
- **Засвар:** Хэрэглэгч өөрийн жинхэнэ нууц үгээ ашигладаг болго. Firebase Auth-д нууц үг найдвартай хадгалагдана — гар аргаар нэгтгэх логикийг бүрэн устга.

### S2. Нууц үг + нууц асуултын хариу Firestore-д ил хадгалагдаж байна
- `User.password`, `securityAnswer1/2` талбарууд ил текстээр Firestore-д.
- **Засвар:** Эдгээр талбарыг Firestore-оос бүрмөсөн хас. Нууц үгийг зөвхөн Firebase Auth удирдана.

### S3. `users` коллекц нийтэд нээлттэй (`allow read: if true`)
- Хэн ч Firestore API-аар бүх хэрэглэгчийн **и-мэйл, утас, нууц үг, нууц асуултыг** шууд татаж авна.
- "Blur" хийдэг логик нь зөвхөн **нүдний хуурмаг** — өгөгдөл DOM/API-д бүрэн байна.
- **Засвар:** Read дүрмийг хэсэгчилнэ. Эмзэг талбарыг (утас, и-мэйл) тусдаа хамгаалалттай уншилтаар өг, эсвэл нийтийн/хувийн талбарыг салга.

### S4. `jobs` / `reviews` — дурын нэвтэрсэн хэрэглэгч засах/устгах эрхтэй
- `allow write: if request.auth != null` — эзэмшигчийн шалгалт байхгүй.
- **Үр дагавар:** Хэн ч бусдын **үнэлгээ, ажлын түүхийг хуурамчаар үүсгэх/устгах** боломжтой. Энэ нь шууд таны платформын итгэлийн системийг устгана.
- **Засвар:** `request.auth.uid == resource.data.employerId` (эсвэл reviewerId) шалгалт нэм.

### S5. Gemini API түлхүүр client талд илчлэгдэж байна
- `NEXT_PUBLIC_GEMINI_API_KEY` нь static JS bundle дотор ил гарна → хэн ч хулгайлж ашиглана (төлбөр танд).
- **Засвар:** AI дуудлагыг server (Firebase Cloud Function / API route) дамжуул. Түлхүүрийг хэзээ ч client-д бүү гарга.

## 3. 🟠 Дунд зэрэг — Архитектур ба Гүйцэтгэл (1-2 сар)

- **Монолит файлууд:** `JobBoard.tsx` (2003 мөр), `db.ts` (1581), `Auth.tsx` (1444), `ProfileView.tsx` (1557). Засвар хийхэд эрсдэлтэй, удаан. → Жижиг hook/component болгон задал.
- **Давхар хадгалалт:** localStorage + Firestore зэрэгцэн. Алдаа, синк зөрүү үүсгэдэг. → Firestore-г цорын ганц эх сурвалж болго.
- **Том bundle:** 477KB + 313KB JS chunk. Firebase SDK хүнд. → `firebase/firestore` lite, динамик import ашигла.
- **Base64 зураг Firestore-д:** Баримтын хэмжээ хавчдаг, удаан. → Firebase Storage руу шилжүүл, зөвхөн URL хадга.
- **Static export + динамик ажил:** Шинэ зар нэмэгдэхэд дахин build хийхгүй бол `/jobs/[id]` хуудас шинэчлэгдэхгүй (SEO). → ISR/SSR эсвэл build hook бодолцох.
- **Commit хийгээгүй 6 файл:** Working tree цэвэр биш. → Ажиллаж буй кодоо commit хийж хамгаал.
- **Зөвхөн 1 тест:** Playwright тест нэг л байна. → Auth, hire, review урсгалд тест нэм.

## 4. 🟡 Дизайн ба UI/UX сайжруулалт

**Сайн талууд:** Цэгцтэй dark glassmorphism, neon өргөлт, mobile-first, Lucide icon, custom scrollbar, зөөлөн анимаци, сайн бодсон монгол SEO дүрэм. Үнэхээр premium харагдана.

**Засах зүйлс:**
- **Өнгөний зөрүү:** `AGENTS.md` нь cyan/emerald/violet гэж заасан ч header-т алтан `#caa03d`, мөн amber/sky/rose холилдоно. → Нэг тод "brand" өнгө тогтоо (жишээ нь emerald).
- **Фонтны зөрүү:** Дүрэмд Inter гэсэн ч бодит код Geist (Geist Sans/Mono) ачаалдаг. → Аль нэгийг сонгож, баримтжуул.
- **Дүрмийн зөрчил:** Header `bg-slate-900` ашигладаг, гэтэл өөрийн дүрэм `#070a13` brand-bg + glass гэж заасан. → Нэгтгэ.
- **Landing хуудас байхгүй:** Нүүр хуудас = шууд зарын самбар. Платформын гол үнэ цэн (итгэл, хариуцлага) тайлбарласан богино hero/танилцуулга байвал шинэ хэрэглэгч, SEO, итгэлд маш чухал.
- **Accessibility:** Бүх апп-д ердөө 1 `aria-label`. Товч/модал дээр label, focus trap, keyboard navigation нэм. 30-50 насны хэрэглэгчдэд contrast, том товч чухал.
